# Claude Code 2.1.170 신규 필드 고도화 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Claude Code 2.1.141~2.1.170에서 추가된 statusline 입력 필드(effort, pr, session_name, thinking)를 파싱·표시하고, settings.json 기반 effort dead code를 제거하며, v2.2.0으로 릴리즈 후 중심 레포(claude-plugins)까지 최신화한다.

**Architecture:** 기존 Section 인터페이스 패턴을 그대로 따라 신규 섹션 4개(Effort, Session, PR, Thinking)를 추가한다. 모든 신규 필드는 optional(absent 가능)이므로 포인터 타입으로 파싱하고, nil이면 빈 문자열을 렌더링해 자동 숨김 처리한다. 배포는 기존 파이프라인(tag push → release.yml → 중심 레포 SHA 동기화)을 사용한다.

**Tech Stack:** Go 1.22, GitHub Actions, gh CLI

**검증 환경:** 이 머신의 Claude Code는 2.1.170, statusline은 `~/.claude/statusline/bin/claude-telemetry` 바이너리를 매 렌더링마다 새 프로세스로 실행하므로 바이너리 교체 즉시 현재 세션에 반영된다.

---

## 배경 (Why)

- Claude Code v2.1.141: statusline JSON 입력에 `effort.level` 추가 (라이브 세션 값, `/effort` 변경 반영)
- Claude Code v2.1.145: `pr.*`, `workspace.repo.*` 추가
- 최근 버전: `session_name`, `thinking.enabled` 추가
- 현재 코드의 `config.ReadEffortLevel()`은 `~/.claude/settings.json`을 읽지만 어떤 섹션도 사용하지 않는 dead code (커밋 92681c0에서 표시 제거됨). 디스크 방식은 세션별 `/effort` 변경을 반영하지 못해 JSON 입력 방식으로 대체해야 한다.
- 플러그인 설치 경로가 버전 포함 캐시 경로(`plugins/cache/.../2.0.0/scripts/run.sh`)로 settings.json에 박혀 있어, 플러그인 업데이트 시 statusline이 깨질 수 있다 → setup이 안정 경로(`~/.claude/statusline/run.sh`)로 복사하도록 개선.

**스코프 제외:** `workspace.repo.*`, `workspace.added_dirs`, `workspace.git_worktree` (gitinfo가 git 명령으로 이미 수집하는 정보와 중복, YAGNI), line 1 adaptive width (후속 과제).

```go
package section

import "github.com/jeongph/claude-telemetry/internal/render"

// EffortSection displays the reasoning effort level (Line 1, Priority 3).
// Claude Code 2.1.141+ 의 effort.level 입력을 사용한다 (라이브 세션 값).
type EffortSection struct{}

func (s *EffortSection) Name() string  { return "effort" }
func (s *EffortSection) Priority() int { return 3 }

func (s *EffortSection) Render(ctx *Context) string {
	eff := ctx.Input.Effort
	if eff == nil || eff.Level == "" {
		return ""
	}
	return ctx.Colors.Dim("↯") + effortColor(ctx.Colors, eff.Level)(eff.Level)
}

// effortColor는 effort 레벨별 색상 함수를 반환한다 (높을수록 강조).
func effortColor(c render.Colors, level string) func(string) string {
	switch level {
	case "low":
		return c.Dim
	case "medium":
		return c.White
	case "high":
		return c.Green
	case "xhigh":
		return c.Yellow
	case "max":
		return c.Red
	default:
		return c.White
	}
}

func (s *EffortSection) Width(ctx *Context) int {
	return render.DisplayWidth(s.Render(ctx))
}
```

```go
package section

import (
	"fmt"

	"github.com/jeongph/claude-telemetry/internal/render"
)

// PRSection displays the open pull request for the current branch (Line 1, Priority 5).
// Claude Code 2.1.145+ 의 pr 입력을 사용한다. PR이 없으면 표시하지 않는다.
type PRSection struct{}

func (s *PRSection) Name() string  { return "pr" }
func (s *PRSection) Priority() int { return 5 }

func (s *PRSection) Render(ctx *Context) string {
	pr := ctx.Input.PR
	if pr == nil || pr.Number == 0 {
		return ""
	}
	out := ctx.Colors.Cyan(fmt.Sprintf("PR#%d", pr.Number))
	switch pr.ReviewState {
	case "approved":
		out += ctx.Colors.Green("✓")
	case "pending":
		out += ctx.Colors.Yellow("●")
	case "changes_requested":
		out += ctx.Colors.Red("✗")
	case "draft":
		out += ctx.Colors.Dim("◌")
	}
	return out
}

func (s *PRSection) Width(ctx *Context) int {
	return render.DisplayWidth(s.Render(ctx))
}
```
